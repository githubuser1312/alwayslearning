BUTTON

Button (a difererncia del input del tipo button) SI sirve para enviar los datos del formulario y SI para recargar la pagina

ATRIBUTOS de los formularios

  1. PLACEHOLDER Con un texto se da una idea al ususario de lo que tiene que introducir.
  2. REQUIERE significa que el campo es obligatorio (CUIDADO, este requisito lo puede eliminar el usuario desde el inspector web, asi que es mejor asegurarse haciendo el campo requiere con Python o JavaScript
  3. READONLYNo se puede modificar por el usuario (a no ser que sea desde el inspector web)
  4. DISABLE significa que lo que tiene el campo no se va a enviar (ni modificar)
  5. MIN-MAX sirven para especificar los valores minimos y maximos que se pueden introducir, se puede especificar uno de los dos
  6. MINLENGTH o MAXLENGTH sirven para especificar lo minimo o maximo de la longitud del texto
  7. SELECTED es como un checked pero para los campos SELECT
  8. AUTOFOCUS es para remarcar el campo donde introducir el dato

Para enviar un formulario se usa METHOD con valor GET o con valor POST. ES MUY IMPORTANTE QUE PARA QUE ESTOS DOS METODOS FUNCIONEN EL FORMULARIO TENGA EL ATRIBUTO NAME

Imágenes

Podemos usar img o picture (siendo el ultimo más nuevo y recomendado). Formatos admitidos: .svg, .jpeg, .gif, .png, .webp (este ultimo es el mas ligero y ademas tiene las ventajas de jpeg, gif y png)

En la página https://whatismyviewport.com podemos comprobar el tamaño de nuestra pantalla y el DPR: DEVICE PIXEL RATIO que es la relacion entre los Pixeles reales y los pixeles lógicos. Por ejemplo un iphone5 tiene pixeles reales 960x640 pero lo pixeles logicos son de 480x320 por eso DPR es 2 en su caso.

Por ejemplo iphoneX tiene 1135x2436 px de resolución pero el viewport da 375x634 px siendo DPR: 3.00 (la relacion entre 2436/634 no es 3 porque el viewpor quita los espacios que ocupan los bordes superior e inferior donde estan los iconos del navegador).

Para usar diferentes tamaños de imagenes dependiendo del tipo de device usaremos el atributo srcset.

<img srcset="/media/prueba.webp, /media/prueba2.webp 2x, /media/prueba3.webp 3x" src="prueba.jpeg" alt="pruebajpeg">

En el ejemplo anterior tenemos prueba para pantallas normales, prueba2 para pantallas moviles con DPR:2 y prueba3 para pantallas moviles con DPR:3. Se usa src para el caso en que no se admiten las webp o el atributo srcset por parte del navegador

Picture, es obligatorio incluir dentro la etiqueta img que es de seguridad, por si no funciona source

En el siguiente ejemplo si la pantalla es menor que 600px se usará prueba2 (para móviles) y si es mayor se usará prueba

<picture> <source srcset="/media/prueba2.webp 2x" media="(max-width:600px)"> <img src="/media/prueba.jpeg" alt=""> </picture>

Audio

Controls pone la imagen de los controles, autoplay reproduce en cuanto se abre la pagina, aunque ultimamente los navegadores no lo estan permitiendo. Para dejar autoplay puesto hay que poner muted para que el altavoz este apagado inicialmente y sea el usuario quien decida escuchar activando el altavoz. Loop reproduce desde el principio cuando termina

<audio> <audio src="/media/sonido.mp3" controls autoplay mute> </audio>

Video

Los atributos son iguales a los de Audio. Añadimos poster que sería la imagen que aparece al principio

<video src="/media/video.mp4" controls autoplay muted loop poster="/media/prueba.jpg"></video>
